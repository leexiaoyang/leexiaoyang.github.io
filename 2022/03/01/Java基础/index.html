Java基础

型变、协变、逆变、不变

型变：用来描述类型转换后的继承关系（即协变、逆变和不变的统称）。比如：List<Orange>是List<Fruit>的子类型吗？答案是No，两者并没有关系，并不能相互读写数据。因此，型变是处理如List<Orange>(List<? extends Orange>)和List<Fruit>子类型关系的一种表达方式。

协变(covariance)：满足条件诸如List<Orange>是List<? extends Fruit>的子类型时，称为协变。

逆变(covariance)：满足条件List<Fruit>是List<? super Orange>的子类型时，称为逆变。

不变(invariance)：表示List<Orange>和List<Fruit>不存在型变关系。

JAVA泛型

参数化类型，将类型当作参数传递给一个类或者是方法。早期Java是使用Object来代表任意类型的，但是向下转型有强转的问题，这样程序就不太安全。

  • 它是一种类型安全检测机制，提供编译期间的类型检测，且具有自动和隐式的类型转化；
  • 泛化代码，代码可以更多的重复利用；

JAVA泛型的用法

  • 泛型方法：将方法的参数类型定义为泛型，在调用的时候接受不同类型的参数。泛型类中的类型参数与泛型方法中的类型参数是没有相应的联系的，泛型方法始终以自己定义的类型参数为准。

  • 泛型类：定义类的时候在类上定义了泛型，传入不同的参数实例化不同的对象，用户想要使用哪种类型，就在创建的时候指定类型。使用的时候，该类就会自动转换成用户想要使用的类型了。

  • 泛型接口：通过在接口名后面添加类型参数的声明部分来实现。泛型接口和泛型类差不多。

类型擦除

泛型只存在源码中，编译器将源码编译成字节码之时，就会把泛型『擦除』，所以字节码中并不存在泛型。最终替换成了 Object。类型擦除的主要目的是避免过多的创建类而造成的运行时的过度消耗。List<A>表示一个类型，再用List<B>表示另一个类型，以此类推，无疑会引起类型的数量爆炸。

当类定义中的类型参数存在限制时，在类型擦除中替换为类型参数的上界或者下界。

具体限定

  • 具体限定只能使用extends关键字做上界限定，不能做下届限定

  • 使用上界限定泛型的类型后，编译时类型擦除就不会转为Object而是转为限定的类。

  • 上界是另一个泛型，泛型还可以继承另一个泛型来限定泛型的类型。

  • T表示一种数据类型，必须实现Comparable接口，且必须可以与相同类型的元素进行比较

通配符限定 、、

  • 通配符限定可以使用extends和super分别来限定泛型的上界和限定泛型的下界，叫有限定通配符

    • 上界通配符 <? extends T> 不能往里存，只能往外取。因为如果支持存入则会破坏类型安全。（生产者只能读取）

    • 下界通配符<? super T> 能存，能取，但取得部分功能被限制，取出来得东西只能放到Object类中。不能保证从中读取到的元素的类型。（消费者只能写入）

    • 频繁set的内容，适合用下界通配符<? super T>，因为上界通配符不支持写入

    • 频繁get的内容，适合用上界通配符 <? extends T>，因为下届通配符会返回Object，每次转换很麻烦。

  • 直接使用<?>，叫无限定通配符

  • <?>被称作无限定的通配符。

    <?>提供了只读的功能，也就是它删减了增加具体类型元素的能力，只保留与具体类型无关的功能。

  • <? extends T>被称作上界通配符。

    代表只接受类型 T 及 T 的子类。`extends` 限制了泛型类型的父类型，所以叫上界。

  • <? super T>被称作下界通配符。

    super 限制了通配符 ? 的子类型，所以称之为下界。

Java重载（Overload）和重写（Override）的区别

重载发生在本类，方法名相同，必须改变参数列表（个数或者参数类型），与返回值、访问修饰符无关。可以改变返回类型，可以改变访问修饰符，可以声明新的或更广的检查异常，方法能够在同一个类中或者在一个子类中被重载。因为调用时不能指定类型信息，编译器不知道你要调用哪个函数。

重写，参数列表必须完全与被重写方法的相同；返回类型必须完全与被重写方法的返回类型相同；访问级别的限制性一定不能比被重写方法的强

  • 重载遵循所谓“编译期绑定”，即在编译时根据参数变量的类型判断应该调用哪个方法。

  • 重写遵循所谓“运行期绑定”，即在运行时根据引用变量所指向的实际对象类型来调用方法。

Java反射创建对象效率高还是通过new创建对象的效率高？

通过new创建对象的效率比较高。通过反射时，先找查找类资源，使用类加载器创建，过程比较繁琐，所以效率较低。

优点： 运行期类型的判断，动态加载类，提高代码灵活度。
缺点： 1,性能瓶颈：反射相当于一系列解释操作，通知 JVM 要做的事情，性能比直接的 java 代码要慢很多。2,安全问题，让我们可以动态操作改变类的属性同时也增加了类的安全隐患。

面向对象编程三大特性

面向对象说到底就是一种思想，任何事物都可以看作是一个对象。

封装

把描述一个对象的属性和行为的代码封装在一个类中，属性用变量定义，行为用方法进行定义。

继承

子类继承父类的特征和行为。子类可以有父类非私有的方法，属性。子类也可以对父类进行扩展，也可以重写父类的方法。缺点就是提高代码之间的耦合性。

多态（编译看左边，运行看右边）

多态是同一个行为具有多个不同表现形态的能力。

Java的多态性体现在两个方面：由方法重载实现的编译时多态和方法重写实现的运行时多态。

  • 编译时多态：在编译阶段，具体调用哪个被重载的方法，编译器会根据参数的不同来确定调用相应的方法。
  • 运行时多态：在编译时，仅仅依据对象所属的类，系统无法确定到底应该执行那个类的方法，只有运行时才能确定。由于子类继承了父类所有的属性（私有的除外），所以使用父类对象的地方，可以用子类对象来代替。一个对象可以通过引用子类的实例来调用子类的方法。

优点：

  1. 提高了代码的维护性（由继承保证）
  2. 提高代码复用性（由继承保证）
  3. 提高了代码的扩展性：新增加的子类不影响已存在类的多态性、继承性以及其他特性
  4. 安全性：向上转型将子类类型隐藏了起来

缺点：

  1. 无法使用子类特有的功能（如要使用就要使用向下转型强制类型转换）

浅拷贝和深拷贝

深拷贝和浅拷贝是只针对Object和Array这样的引用数据类型的。

  • 浅拷贝只复制指向某个对象的指针，而不复制对象本身，新旧对象还是共享同一块内存。对基本数据类型进行值传递。
  • 深拷贝会另外创造一个新的对象，并复制其内容，新对象跟原对象不共享同一块内存，修改新对象不会影响到原对象。

clone() 方法

clone() 方法实则是真的创建了一个新的对象。如果一个对象内部只有基本数据类型，那用 clone() 方法获取到的就是这个对象的深拷贝，而如果其内部还有引用数据类型，那用 clone() 方法就是一次浅拷贝的操作。

String 类型很特殊，它是不可变类型。String类型的引用将会指向常量。进行克隆后，得到一个新的对象，如果该对象的string变量重新赋值，那么只会有这个string 引用变量指向一个新的内存区域，对原来对象中的string变量不会有影响。

如何实现深拷贝

  • 序列化（serialization）这个对象，再反序列化回来，就可以得到这个新的对象，无非就是序列化的规则需要我们自己来写。
  • 继续利用 clone() 方法，可以对其内部的引用类型的变量，再进行一次 clone()。

equals与 == 的区别

==

  • 基本类型：比较的是值是否相同；（byte,short,char,int,long,float,double,boolean等基本类型）
  • 引用类型：比较的是在内存中的存放地址是否相同；

equals

  • 引用类型：默认情况下，比较的是内存地址值。
  • 只不过 String 和 Integer 等重写了 equals 方法，把它变成了值比较。
  • String类中被复写的equals()方法其实是比较两个字符串的内容。

什么时候需要重写equals与 hashcode

hashcode：元素的地址转换成一个整数返回

只有当类需要作为key放在HashTable和HashMap等这些具有Hash结构的集合中才需要重写hashcode。因为 hash结构的集合存储的是不重复的对象，依据 hashCode 和 equals 进行判断，所以 hash结构的集合存储的对象必须重写 hashCode 和 equals。

为什么equals相等hashcode必相等

如果只重写了 equals 方法，两个对象 equals 返回了true，集合是不允许出现重复元素的，只能插入一个。此时如果没有重写 hashCode 方法，那么就无法定位到同一个位置，集合还是会插入元素。这样集合中就出现了重复元素了。那么重写的equals方法就没有意义了。

StringBuilder是线程不安全的，效率较高；而StringBuffer是线程安全的，效率较低。

抽象类能使用 final 修饰吗

不能，定义抽象类就是让其他类继承的，如果定义为 final 该类就不能被继承，这样彼此就会产生矛盾，所以 final 不能修饰抽象类

接口和抽象类区别

  • 接口中没有构造方法，而抽象类中可以有。
  • 接口中的方法修饰符默认是public abstract，抽象类可以是任意访问修饰符。
  • 抽象类有的变量默认是 public static final 修饰，而抽象类中的变量无限制。
  • 接口中不能含有静态方法（java8取消限制）以及静态代码块，而抽象类可以有。
  • 接口可以多实现 implement， 抽象类只能单继承 extends ，一个类如果继承抽象类，要么实现它的抽象方法，要么也定义为一个抽象类。
  • 接口是不可以被实例化，抽象类也不可以被实例化。

  • default 方法 （实现类可以选择重写，也可以不重写）

  • static 方法 （接口名.static方法名调用，切记实现类不能继承static方法）

final、finally、finalize 有什么区别

  • final可以修饰类、变量、方法，修饰类表示该类不能被继承、修饰方法表示该方法不能被重写、修饰变量表示该变量是一个常量不能被重新赋值。
  • finally一般作用在处理异常的try-catch代码块后，表示不管是否出现异常，该代码块都会执行，一般用来存放一些关闭资源的代码。
  • finalize是Object类的一个方法，而Object类是所有类的父类，该方法一般由垃圾回收器来调用，当我们调用System的gc()方法的时候，垃圾回收器会调用finalize()。一般用来堆那些即将被垃圾回收器标记的对象重新赋值，从而达到逃生的目的。

四种访问修饰符

  • private：类中限定为private的成员，只能被这个类本身访问。如果一个类的构造方法声明为private,则其它类不能生成该类的一个实例。
  • default：类中不加任何访问权限限定的成员属于缺省的（default）访问状态，可以被这个类本身和同一个包中的类所访问。
  • protected：类中限定为protected的成员，可以被这个类本身、它的子类（包括同一个包中以及不同包中的子类）和同一个包中的所有其他的类访问。
  • public：类中限定为public的成员，可以被所有的类访问。
