关键字 Reified

使抽象的东西更加具体或真实

  • 通过添加类型传递简化泛型参数
  • 使用 reified，简化泛型参数和保证 as? 类型转换安全性
  • 使用 reified，可以实现不同的返回类型函数重载

out 和 in

  • 使用关键字 out 来支持协变，等同于 Java 中的上界通配符 ? extends。
  • 使用关键字 in 来支持逆变，等同于 Java 中的下界通配符 ? super。

*符号

前面讲到了 Java 中单个 ? 号也能作为泛型通配符使用，相当于 ? extends Object。 它在 Kotlin 中有等效的写法：* 号，相当于 out Any。比如你的类型定义里是 out T : Number 的，那它加上 <*> 之后的效果就不是 out Any，而是 out Number。

伴生对象Companion

Companion object 的出现是为了解决 Java static 方法的反面向对象（Anti-OOP）的问题。static 方法没有面向对象的消息传递和延迟绑定特性。提出了伴生对象这个概念来代替 static 方法。companion 伴生对象是一个对象，它在类初始化时被实例化。因为伴生对象不再是类的 static 方法，而是某个类的实例化对象，所以它可以声明接口，里面的方法也可以被重写，具备面向对象的所有特点。

  • 即使伴⽣对象的成员看起来像其他语⾔的静态成员，在运⾏时他们仍然是真实对象的实例成 员，我们可以使⽤ @JvmStatic
  • 注解，你可以将伴⽣对象的成员⽣成为真正的静态⽅法和 字段。

伴生对象实现接口

伴生对象跟其他对象声明一样，可以实现接口。其调用方式就是直接使用容器类来调用接口实现，就像容器类实现了相关接口一样。

object关键字

本身就是一个对象了，不需要像类一样要创建实例，可以直接调用方法。这就被称作对象声明。可以用于创建单例对象。

对象表达式（匿名内部类）

取代Java的匿名内部类的写法。与Java的匿名内部类只能扩展一个类或实现一个接口不同，Kotlin中匿名对象可以实现多个接口或者不实现接口。
